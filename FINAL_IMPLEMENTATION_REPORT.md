- **実装ファイル**:

```
✓ 香: 前に何マスでも（障害物で停止）

✓ 飛: 上下左右（障害物で停止）
✓ 玉: 8方向各1マス
```

### ✅ 2. 駒の成り判定と必須成り
- **成り判定**: 敵陣（後手なら1-3段目）内での移動で成り選択可能
- **必須成り**: 歩・香が敵陣最終段に移動時は必須成り
- **実装ファイル**:
  - `lib/services/legal_move_validator.dart` - 必須成り検出（120行）
  - `lib/services/shogi_game_validator.dart` - 成り検証（150行）

**検証済み内容**:
```
✓ 成駒は元の駒に戻さない（駒台に落とすときも成駒のまま）
✓ 歩を最終段にドロップできない
✓ 香を最終段にドロップできない
✓ 敵陣内での成り選択可能
✓ 必須成りの時は強制成り
```

### ✅ 3. 王手（チェック）検出と王手放置禁止
- **王手判定**: 敵の任意の駒が王を攻撃しているか検出
- **王手放置防止**: 移動後に王手状態になる手を禁止
- **実装ファイル**:
  - `lib/services/check_detector.dart` - 王手検出エンジン（90行）
  - `lib/services/legal_move_validator.dart` - 王手放置防止（120行）

**検証済み内容**:
```
✓ 自分の王が敵に攻撃されている → 王手
✓ 王手を放置してはいけない → 自動チェック
✓ 王は敵に攻撃されるマスに移動不可
✓ 王手を放置する手は合法手から除外
```

### ✅ 4. 詰み（チェックメイト）判定
- **詰み判定**: 王手状態で全ての手が王手を解除できない
- **実装ファイル**:
  - `lib/services/legal_move_validator.dart` - 詰み判定（`isCheckmate()`）

**検証済み内容**:
```
✓ 王手状態である
✓ 王を動かせない
✓ 敵の攻撃駒を取れない
✓ 敵の攻撃駒の進路を塞げない
→ 詰み判定 ✓
```

### ✅ 5. 駒台管理（捕獲駒の保管）
- **駒台機能**: 捕獲した駒を種類別・色別に管理
- **ドロップ機能**: 駒台の駒を盤面に戻すことができる
- **実装ファイル**:
  - `lib/services/captured_piece_manager.dart` - 駒台管理システム（130行）

**検証済み内容**:
```
✓ 敵の駒をキャプチャ → 駒台に追加
✓ 成駒をキャプチャ → 元の駒に戻す
✓ 駒台の駒をドロップ → 盤面に配置
✓ 先手・後手の駒台を個別管理
```

### ✅ 6. ドロップ時の禁止事項
- **二歩禁止**: 同じ筋に2つの歩がある場合、歩をドロップ不可
- **打ち歩詰め禁止**: ドロップ歩で直ちに詰みになる場合は不可
- **位置制限**: 歩・香は敵陣最終段、桂は最終2段にドロップ不可
- **実装ファイル**:
  - `lib/services/drop_move_validator.dart` - ドロップ検証（150行）

**検証済み内容**:
```
✓ 二歩チェック: 筋内の歩を検索
✓ 打ち歩詰めチェック: テンポラリボードで詰み判定
✓ 歩の位置チェック: 敵陣最終段を除外
✓ 香の位置チェック: 敵陣最終段を除外
✓ 桂の位置チェック: 敵陣最終2段を除外
✓ ドロップ先がクリア: 空マスのみ許可
```

### ✅ 7. 千日手（トリプル繰り返し）判定
- **千日手**: 同じ局面が4回繰り返されると引き分け
- **Zobristハッシング**: 局面を効率的に比較
- **実装ファイル**:
  - `lib/services/draw_detector.dart` - 千日手検出（110行）

**検証済み内容**:
```
✓ 局面をハッシュ化
✓ ハッシュ履歴を記録
✓ 同じハッシュが4回出現 → 千日手
✓ 駒台情報も含めたハッシュ
```

### ✅ 8. ゲーム状態の統合管理
- **統一検証エンジン**: 全ての検証を一元管理
- **ゲームステータス**: 通常/王手/詰み/千日手/ステイルメイト
- **実装ファイル**:

  - `lib/services/shogi_game_state.dart` - ゲーム状態管理（200行）

## 🚀 使用方法

### 1. ゲームの初期化
```dart
import 'package:flutter/material.dart';
import 'lib/services/shogi_game_state.dart';

void main() {
  runApp(
    ChangeNotifierProvider(
      create: (_) => ShogiGameState(),
      child: MyApp(),
    ),
  );
}
```

### 2. 駒の選択
```dart
// ユーザーが盤面をタップ
gameState.selectPiece(3, 6);
// → possibleMoves に合法手が自動生成
```

### 3. 駒の移動
```dart
gameState.movePiece(3, 6, 4, 6, shouldPromote: false);
// → 王手放置がないか自動確認
// → 成り判定も自動実行
// → ゲーム状態も自動更新
```

### 4. ゲーム状態の確認
```dart
print(gameState.gameStatus);   // normal / inCheck / checkmate / draw_threefoldRepetition
print(gameState.gameMessage);  // "先手のターン" / "王手です！" など
print(gameState.capturedPieces.getCount(PieceType.pawn, true)); // 先手の歩の数
```

### 5. ゲーム終了判定
```dart
if (gameState.isGameOver()) {
  print('ゲーム終了: ${gameState.gameStatus}');
}
```
